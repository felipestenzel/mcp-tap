- **Date**: 2026-02-25
- **Issue**: #112
- **Status**: `done`
- **Branch**: `fix/2026-02-25-release-v069`
- **Priority**: `high`

## Problem

PR #111 is merged in `main`, but no package release has been cut yet. Users cannot consume the new GitHub maturity fallback/transparency behavior from registries.

## Context

- Current version in `main`: `0.6.8`
- PR #111 introduced improvements in GitHub metadata auth fallback and degraded maturity transparency
- Release workflow requires: version bump, validation, PR merge, tag/release publish, registry verification, smoke real

## Root Cause

Release publishing is a separate explicit step after feature PR merge.

## Solution

Completed on 2026-02-25:

1. Bumped versions to `0.6.9`:
   - `pyproject.toml`
   - `npm-wrapper/package.json`
   - `uv.lock` package entry
2. Opened and merged PR #113 (`fix/2026-02-25-release-v069`) with green CI.
3. Published GitHub release/tag `v0.6.9`.
4. Verified registry availability:
   - PyPI JSON API reports latest version `0.6.9`.
   - npm registry reports latest version `0.6.9`.
5. Executed real smoke checks from published artifacts:
   - `uvx --refresh --index-url https://pypi.org/simple --from mcp-tap==0.6.9 python -c "import mcp_tap; print(mcp_tap.__version__)"`
     -> `0.6.9`
   - `uvx --refresh mcp-tap@0.6.9 --help` -> exit code `0`
   - `npx -y mcp-tap@0.6.9 --help` -> exit code `0`
